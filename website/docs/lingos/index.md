# Lingos / LingOS

This page documents the current public naming split to prevent drift.

## Status block

- **Public docs status**: Public
- **Implementation status**: Clarify before broad claims
- **Naming law**: [See governance rule](/governance/naming-law)

## Working rule

Use one of these and keep it consistent:

- **Lingos**: symbolic, glyph-native operating layer
- **LingOS**: public framework/product naming where applicable

If the project uses both, the difference must be stated explicitly.

## What this layer is for

- conversational interfaces
- symbolic execution grammar
- memory-aware interactions
- observability hooks into MirrorDNA
- governed actions through ActiveMirrorOS and policy rails

## What this page avoids

This page deliberately avoids unverified package-install and provider examples until they are linked to a repo, release, or package artifact.

## Proof rail

Add at least one of these before expanding claims:

- public repo
- package registry link
- CLI install proof
- demo video
- running example
- tagged release
