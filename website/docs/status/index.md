# Status

MirrorDNA docs now separate **what exists**, **what is public**, and **what is planned**.

## Status vocabulary

- **Concept**: idea level
- **Spec**: defined in docs/spec, not necessarily implemented
- **Prototype**: partial working implementation
- **Internal**: real but not yet public-facing
- **Public**: visible repo/site/artifact exists
- **Shipping**: actively usable and evidenced
- **Deprecated**: no longer current

## Rule

No major component page should imply shipping status without:
- a visible status block
- a proof rail
- a link to supporting artifacts

See [Component Status](/status/components).
